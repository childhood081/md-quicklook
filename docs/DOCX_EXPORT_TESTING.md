# DOCX Export Testing

本文档用于手动验收 md-quicklook 的 Markdown 导出 Word 功能。

## 测试步骤

1. 打开 `docs/test-docx-export.md`。
2. 点击工具栏里的 `Word` 导出按钮。
3. 用 Word / WPS / Pages 打开生成的 `docs/test-docx-export.docx`。
4. 对照 `docs/test-docx-export.md` 原文检查内容是否完整。
5. 记录每类 Markdown 元素的支持状态。

## 完全支持

- H1-H4 标题层级。
- 普通中文段落、英文段落、中英文混排。
- 加粗、斜体、行内代码。
- 基础无序列表和有序列表。
- 普通引用段落。
- 围栏代码块，导出为等宽文本样式。
- Markdown 表格，包括多个表格。
- 水平分割线。

## 降级支持

- `http`、`https`、`mailto` 链接优先导出为超链接。
- 相对链接导出为普通文本，避免 Word 中生成不稳定的外部关系。
- 图片导出为 `[Image: alt]` 占位文本。
- 表格单元格内的加粗、斜体、行内代码会尽量保留；复杂块级内容会降级。

## 暂不支持

- 嵌套列表和多段列表项的完整 Word 结构。
- 引用块中的列表、表格、代码块等复杂块级内容。
- 图片嵌入和复杂图片排版。
- 复杂 Word 样式、页眉页脚、批注、修订、文本框。

## 边界用例

- 空文档：应能导出空白 Word 文档，不应报错。
- 只有标题：应导出对应标题层级。
- 只有表格：应导出 Word 表格。
- 只有代码块：应导出等宽文本代码段。
- 当前 Markdown 已修改但未保存：导出的 Word 应使用编辑器内存中的最新内容。

## 回归检查

- 导出 Word 后，阅读模式内容不应变化。
- 导出 Word 后，源码模式内容不应变化。
- 导出 Word 后，保存状态不应被错误改写。
- Excel 导出仍应可用，且不会受到 Word 导出的影响。
