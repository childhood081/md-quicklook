# Windows Menu Testing

本文档用于验证 Windows 原生菜单和前端编辑器行为是否正确联动。macOS 可按同样步骤做回归测试。

## 前置条件

1. 启动应用并打开一个 `.md` 或 `.markdown` 文件。
2. 准备一段包含标题、正文、列表、表格和 Front Matter 的 Markdown。
3. 分别在阅读模式、编辑模式、源码模式执行菜单操作。

## 编辑菜单

### 撤销 / 重做

1. 切换到源码模式。
2. 输入一段文字。
3. 点击 `编辑 -> 撤销`，确认刚输入的内容被撤销。
4. 点击 `编辑 -> 重做`，确认内容恢复。
5. 使用 `Ctrl+Z` / `Ctrl+Y` 再测试一次。
6. 切换到编辑模式，重复修改、撤销、重做。
7. 切换到阅读模式，点击撤销 / 重做，应显示“当前模式不支持此操作”，不能报错。

### 剪切 / 复制 / 粘贴 / 全选

1. 源码模式中选中一段文本。
2. 测试 `剪切`、`复制`、`粘贴`、`全选` 菜单。
3. 使用 `Ctrl+X`、`Ctrl+C`、`Ctrl+V`、`Ctrl+A` 验证快捷键。
4. 编辑模式中重复上述测试。
5. 阅读模式中复制和全选可以使用系统选择能力；剪切 / 粘贴应降级提示，不能报错。

### 查找

1. 源码模式中点击 `编辑 -> 查找`。
2. 确认 CodeMirror 搜索面板打开。
3. 输入关键词，确认可定位匹配内容。
4. 阅读模式中点击查找，应弹出简单查找输入框并定位页面内容。
5. 编辑模式第一版不提供专用查找面板，应提示当前模式暂不支持查找，不能报错。

## 视图菜单

### 放大 / 缩小 / 实际大小

1. 点击 `视图 -> 放大`，确认文档区域放大。
2. 连续点击，确认最大约为 200%。
3. 点击 `视图 -> 缩小`，确认文档区域缩小。
4. 连续点击，确认最小约为 50%。
5. 点击 `视图 -> 实际大小`，确认恢复 100%。
6. 确认 Markdown 原文没有变化，保存状态不因缩放改变。

### 全屏

1. 点击 `视图 -> 全屏`。
2. 确认窗口进入全屏。
3. 再次点击或使用系统退出全屏方式，确认可以恢复。
4. 如果系统或权限拒绝，全屏失败应显示错误提示，不能导致白屏或崩溃。

## 模式切换联动

1. 在编辑模式修改内容。
2. 用菜单切换到源码模式，确认内容同步。
3. 在源码模式撤销 / 重做，确认 Store content 同步。
4. 切回阅读模式，确认渲染为最新内容。
5. 等待自动保存，关闭重开确认内容保留。
